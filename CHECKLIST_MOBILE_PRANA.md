# 📱 Checklist de Revisão Mobile Prana

## 1. Revisão de Fluxos Mobile
- [ ] Chat: interface otimizada, input fixo, envio rápido
- [ ] Tarefas: criação/edição fácil, visualização compacta
- [ ] Dashboard: widgets e cards adaptados para toque
- [ ] Planner/Calendário: navegação por dias/semanas, fácil adicionar eventos
- [ ] Menu/Drawer: acesso rápido às principais áreas
- [ ] Bottom Nav: abas intuitivas, sem rolagem excessiva

## 2. Usabilidade e UX
- [ ] Ações principais a 1-2 toques de distância
- [ ] Botões e áreas de toque grandes o suficiente
- [ ] Fontes legíveis em telas pequenas
- [ ] Animações suaves, sem travamentos
- [ ] Gestos: swipe para drawer, pull-to-refresh, etc
- [ ] Feedback visual (toasts, loaders, confirmações)

## 3. Performance
- [ ] Testar em dispositivos reais (Android/iOS)
- [ ] Listas e navegação rápidas
- [ ] Sem delays perceptíveis em transições

## 4. Integração Nativa
- [ ] Push notifications (Capacitor + Firebase/APNs)
- [ ] Splash screen e ícones nativos
- [ ] Permissões (notificações, arquivos, etc)
- [ ] Testar build nativo (Android/iOS)

## 5. Testes de Usabilidade
- [ ] Testes reais com usuários
- [ ] Identificar pontos de fricção
- [ ] Ajustar fluxos conforme feedback

## 6. Documentação
- [ ] Documentar diferenças de UX mobile vs desktop
- [ ] Checklist de boas práticas para novos componentes mobile

---

**Como usar:**
- Marque cada item conforme for revisado/implementado.
- Priorize fluxos críticos (chat, tarefas, planner).
- Teste em dispositivos reais sempre que possível.
- Atualize a checklist conforme novas necessidades surgirem.
